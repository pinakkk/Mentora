1. Vision & Value Proposition

The Problem

Students receive generic, stateless, reactive career guidance. Every existing tool (VMock, Jobscan, ChatGPT) falls into one of two traps:

  • The "Tool" Trap: Does ONE thing (resume scoring, speech analysis) but doesn't connect the dots.
  • The "Chatbot" Trap: Answers questions but never INITIATES, never REMEMBERS, never DECIDES.

No tool acts like a real mentor who: remembers your entire history, proactively checks on your progress, adjusts plans based on performance, holds you accountable, makes strategic decisions, and escalates when you're in trouble.

The Solution: PlaceAI

An Agentic AI Career Coach powered by 6 specialized agents that collaborate through shared memory. The AI doesn't just respond — it thinks, decides, and acts autonomously.

  • Extraction: Claude Haiku 4.5
  • Embeddings: OpenAI text-embedding-3-small
  • SDK: Vercel AI SDK

Why Custom Agents?

Instead of LangChain or CrewAI, we build custom agent loops. This provides total control over reasoning, hand-offs, and shared memory, resulting in a system that is highly debuggable and explicitly tailored for educational coaching rather than general-purpose tasks.

5. Memory & RAG Architecture

The intelligence of PlaceAI relies entirely on a robust, multi-tiered memory system allowing agents to persist context across weeks of interaction.

Layer 1: Working Memory (Ephemeral)

Current conversation messages, active tool call results, and injected context. Lives only for the duration of a single request.

Layer 2: Episodic Memory (Recent)

Last 5 conversation summaries, hot-cached facts, and recent task status. Stored in Upstash Redis with a 7-day TTL.

Layer 3: Semantic Memory (Long-term)

All extracted facts (goals, skills, struggles, preferences) stored as vector embeddings in Supabase pgvector for semantic search and injection.

Memory Injection Protocol

SYSTEM PROMPT = base_agent_prompt + """

6. Data & API Infrastructure

Core pgvector Schema

CREATE TABLE memory_facts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id UUID REFERENCES students(id),
  fact TEXT NOT NULL,
  category TEXT NOT NULL CHECK (category IN ('goal','skill','struggle',...)),
  embedding vector(1536),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX ON memory_facts USING hnsw (embedding vector_cosine_ops);

API Route Structure (Next.js)

  • POST /api/chat # Streaming SSE agent loop
  • POST /api/resume/analyze # AI extraction
  • GET /api/students/me/memory # Coach's notebook data
  • POST /api/agents/proactive # Cron triggered nudges
